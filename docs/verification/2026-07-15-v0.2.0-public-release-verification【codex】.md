# One Holy Bible v0.2.0 public release verification

Date: 2026-07-15 (Asia/Shanghai)

## Data snapshot

- 66 book packages.
- 31,102 CUV verses and 31,102 KJV verses.
- 10,963 explanatory text cards.
- 2,705 image cards referencing 2,515 unique PNG asset descriptors.
- Asset URLs use HTTPS and are pinned to asset repository commit `ad75722ffeb6355dfa77f6b42496f0dd1421a2b2`.
- The checked-in asset manifest digest is recorded in `PUBLIC_RELEASE.json`.

## Automated verification

The public checkout runs the following gates without private source inputs:

```bash
npm run validate:public-data
npm run validate:public-repository
npm run validate:public-release
npm test
npx tsc --noEmit
npx tsc -p tsconfig.node.json --noEmit
npm run build
npm audit --omit=dev
```

The data validator reconciles all book byte counts and SHA-256 values, canonical scripture coordinates, search-index entries, text-card and image-card schemas, asset descriptors, release totals, and public URL policy. Repository hygiene rejects local paths, private service URLs, credentials, development-only directories, and tracked files over 50 MiB.

## Separation guarantee

The public snapshot is maintained independently from the in-progress development workbench. The release contains no raw development resource directory and no private source import path; image binaries are served from the separate asset repository instead of being copied into ordinary application Git history.
