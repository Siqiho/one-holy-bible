# Security policy

## Supported versions

Security updates are currently provided for the latest published release.

## Reporting a vulnerability or sensitive-data issue

Please do not open a public issue for vulnerabilities, exposed credentials, private paths, or content that may have been published without permission. Use GitHub's **Security** tab and choose **Report a vulnerability** to contact the maintainer privately.

Include the affected version, a concise reproduction, expected impact, and any relevant logs with secrets and personal data removed. For a content-rights concern, identify the affected record or file and the basis for the concern without reproducing restricted material.

The maintainer will acknowledge a report when possible, investigate it, and coordinate remediation or removal before public disclosure.
